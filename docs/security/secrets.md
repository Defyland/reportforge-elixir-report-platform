# Secrets

## Current Secrets

| Secret | Purpose | Current source |
| --- | --- | --- |
| Tenant API key | Authenticates API clients | Generated by the app, digest stored in PostgreSQL |
| `SIGNING_SECRET` | Signs artifact download URLs | Environment variable |
| `SIGNING_SECRET_FILE` | File-based signing secret for container platforms | Mounted file path |
| `DATABASE_URL` credentials | Connects to PostgreSQL | Environment variable |
| `REPORT_FORGE_S3_ACCESS_KEY_ID` | S3-compatible storage access | Environment variable |
| `REPORT_FORGE_S3_SECRET_ACCESS_KEY` | S3-compatible storage secret | Environment variable |
| `REPORT_FORGE_S3_SECRET_ACCESS_KEY_FILE` | File-based S3-compatible secret | Mounted file path |

## Rules

- API keys are only shown in clear text at creation time.
- API-key digests are stored with SHA-256 hashing in the current slice.
- Signing secrets must be different across environments.
- Secret values must not appear in logs, lifecycle events, metrics, or traces.
- Secret files should be mounted read-only by the runtime platform.

## Future Hardening

- Managed secret store integration.
- Signing-secret key versions and rotation windows.
- Emergency invalidation for leaked signed URLs.
- API-key rotation procedures and operator tooling.
- Dependency on platform audit logs for secret reads.
